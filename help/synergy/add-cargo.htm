Add Cargo

The Add Cargo option on the Loadable Study page allows the user to add a maximum of seven cargoes for loading operations during the cargo nomination stage. 

To add cargo for loading, perform the steps given below:

  1. Navigate to CARGO PLANNING > Loadable Study. 

    By default, the page displays the loadable studies created against the current active voyage.

    To view and edit the loadable study created against another voyage, select the required option from the Voyage No drop-down list. 
  2. Select the required loadable study from the Loadable Study List table. 

    The respective CARGO NOMINATION tab appears. 
  3. Click Add Cargo. 

    Alternatively, duplicate an existing cargo if required. For more details, see Duplicate Cargo.
  4. A new row appears at the end of the cargo list table.

    Update the fields in the row as per the table given below. 
    Field Name Type Mandatory Description

    Priority 

    Drop-down

    Yes

    This field allows the user to add the cargo based on a priority basis.

    The user is allowed to select from a priority range of one to seven. 

    Color

    Color Picker

    Yes

    This field allows the user to choose a color for the new cargo.

    The color of each cargo must be unique. This color is used to represent the respective cargo in the stowage plan. 

    Cargo

    Drop-down

    Yes

    This field allows the user to select the required cargo from the list.

    ABV

    Text

    Yes

    This field allows the user to add the abbreviation for the selected cargo. 

    The abbreviation for each cargo must be unique. 

    Loading Port

    Pop-up

    Yes

    This field allows the user to select one or multiple loading ports for cargo loading and enter the loadable quantity of the selected cargo planned at each port.

    For more information, see Select Port. 

    Quantity

    Numeric

    Yes

    This field populates the total quantity of cargo based on the data entered in the LOADING PORT field. 

    Maximum Tolerance

    Numeric

    Yes

    This field allows the user to enter the maximum tolerance limit for the selected cargo. 

    The maximum tolerance must be less than or equal to 25. 

    Minimum Tolerance

    Numeric

    Yes

    This field allows the user to enter the minimum tolerance limit for the selected cargo. 

    The minimum tolerance must be less than or equal to zero. 

    API EST

    Numeric

    Yes

    This field allows the user to enter the estimated API value for the cargo at the selected port. This is the standard measurement used to calculate the density of cargo. 
    The application populates the default API value for the selected cargo. The user can edit the data if required. 

    The API EST must be greater than or equal to eight.  
    Click the History icon () to view the API and Temperature history of the selected cargo. 

    For more information, see API and Temperature History.

    TEMP EST

    Numeric

    Yes

    This field allows the user to enter the estimated temperature of the cargo. 
    The application populates the default temperature of the selected cargo. The user can edit the data if required. 

    The TEMP EST must be greater than or equal to 40 degree Fahrenheit. 

    Click the History icon () to view the API and Temperature history of the selected cargo.

    For more information, see API and Temperature History.

    Segregation 

    Drop-down

    Yes

    This field allows the user to select the mode of segregation for the cargo loading or discharging. 

    The available options are:

    • Single - Select this option if the loading or discharging operation is carried out using a single valve pipeline. 
    • Double - Select this option if the loading or discharging operation is carried out using a double valve pipeline. 
  5. Click the Save icon () to save the changes. 

The application allows the user to search the cargo list table based on the field names.
