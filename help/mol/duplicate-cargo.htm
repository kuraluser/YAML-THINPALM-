Duplicate Cargo

Perform the following steps to duplicate an existing cargo:

  1. From the menu bar, click CARGO PLANNING > Loadable Study. 

    By default, the page displays the loadable studies created against the active voyage.

    To view and edit the loadable study created against another voyage, from the Voyage No drop-down, select the required option. 
  2. From the Loadable Study List table, select the required loadable study. 

    The respective Cargo Nomination tab appears. 
  3. From the cargo list table, click  > Duplicate to duplicate the cargo.
     
    A new row appears at the end of the cargo list table, populating the details of the selected cargo. 
  4. The table displays the following details and the user can edit the fields if required.
    Field Type Mandatory Description

    Priority 

    Drop-down

    Yes

    Displays the priority mapped against the existing cargo. The user can edit the priority value if required. The user can select from a priority range of one and seven. 

    Color

    Color Picker

    Yes

    Displays the color of the existing cargo. The color of each cargo must be unique. This color is used to represent the respective cargo in the stowage plans. 

    Cargo

    Drop-down

    Yes

    Displays the existing cargo name. 

    ABV

    Text

    Yes

    Displays the abbreviation for the selected cargo. The abbreviation for each cargo must be unique. 

    Loading Port

    Pop-up

    Yes

    Displays the loading ports and the planned loadable quantity mapped against the existing cargo at each port. The user can edit the loading port details if required. For more information, see Select Port. 

    Quantity

    Numeric

    Yes

    Displays the total quantity of cargo based on the data entered in the Loading Port field. 

    Maximum Tolerance

    Numeric

    Yes

    Displays the maximum tolerance limit for the selected cargo. The user can edit the data if required. The maximum tolerance must be less than or equal to 25. 

    Minimum Tolerance

    Numeric

    Yes

    Displays the minimum tolerance limit for the selected cargo. The user can edit the data if required. The minimum tolerance must be less than or equal to zero. 

    API EST

    Numeric

    Yes

    Displays the estimated API value for the cargo at the selected port. API value is the standard measurement used to calculate the density of cargo. 
    The application populates the default API. The user can edit the data if required.

    The API EST must be greater than or equal to eight.  
    Click  to view the API and Temperature history of the selected cargo. 

    For more information, see API and Temperature History.

    TEMP EST

    Numeric

    Yes

    Displays the estimated temperature of the cargo. 
    The application populates the default temperature. 

    The user can edit the data if required.

    The TEMP EST must be greater than or equal to 40 degree Fahrenheit. 

    Click  to view the API and Temperature history of the selected cargo.

    For more information, see API and Temperature History.

    Segregation 

    Drop-down

    Yes

    Displays the segregation mode marked against the selected cargo for loading and discharging operations. 

    The user can edit the data if required.

    The available options are:

    • Single - Select this option if the loading or discharging operation uses a single valve pipeline. 
    • Double - Select this option if the loading or discharging operation uses a double valve pipeline. 
  5. Click  to save the changes.
